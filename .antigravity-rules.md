# Antigravity (AIエージェント) 動作ルール
# テーマ: AIのトークン制限・コンテキスト肥大化の回避 (Token Optimization & Context Management)

今後、Antigravity（AIエージェント）がタスクを実行する際は、自身のトークン（コンテキストウィンドウ）の浪費や、LLMのAPIレート制限に引っかかることを防ぐため、以下のルールを厳格に遵守すること。

## 1. ファイル読み取り・コマンド出力の最小化 (Optimize File Reading & Output)
- **部分読み取りの徹底**: ファイルを閲覧（`view_file`）する際は、全体を一度に読み込まず、必ず `StartLine` と `EndLine` を指定して必要な部分のみを抽出すること。
- **ターミナル出力の制限**: ターミナルコマンドを実行する際、出力が大量になると予想される場合（`npm install`, 長いログ出力など）は、出力を捨てるか（`> /dev/null` または `Out-Null`）、必要な行数に絞ること。

## 2. ブラウザサブエージェントの節約 (Minimize Browser Subagent Usage)
- **APIの優先**: ブラウザのUI操作（スクリーンショットやDOM解析）は非常に大量のトークンを消費する。そのため、API経由で実行可能な処理（データ収集、投稿操作など）は、可能な限りブラウザ操作よりAPIを優先すること。
- **タスクの細分化**: ブラウザサブエージェントを起動せざるを得ない場合でも、漠然とした指示を与えず、「特定の入力欄に入力し、このボタンを押して結果を返す」というように極力短時間・少ない手数で終わるよう細分化すること。

## 3. 会話セッションのリセット提案 (Session Management)
- **区切りでのリフレッシュ推奨**: 1つの会話セッション内でコード修正、ブラウジング、画像の入出力などを大量に行い、タスクの「大きな区切り」を迎えた場合は、コンテキスト肥大化を防ぐため、ユーザーに対して **「動作を軽くするために、ここで一度『新しいチャット（New Conversation）』を開始することをお勧めします」** と自主的に提案すること。
- **不要な要約の省略**: 毎回の発言で、それまでの経緯を長々と再要約することは避け、簡潔に「何を行ったか、次はどうするか」だけを伝えること。
